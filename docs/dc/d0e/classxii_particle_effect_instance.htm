◆ AddWindSampleLocation()

xiiInt32 xiiParticleEffectInstance::AddWindSampleLocation ( const xiiVec3 &  vPos)

Adds a location where the wind system should be sampled.

Particle behaviors can't sample the wind system directly, because they are updated in parallel and this can cause crashes. Therefore the desired sample locations have to be cached. In the next frame, the result can be retrieved via GetWindSampleResult() with the returned index.

Only a very limited amount of locations can be sampled (4) across all behaviors.

◆ GetTotalEffectLifeTime()

xiiTime xiiParticleEffectInstance::GetTotalEffectLifeTime ( ) const
inline

Returns the total (game) time that the effect is alive and has been updated.

Use this time, instead of a world clock, for time-dependent calculations. It is mostly tied to the world clock (game update), but additionally includes pre-simulation timings, which would otherwise be left out which can break some calculations.
